# 남은 계절 — 메멘토 모리 모바일 웹

죽음을 인식할 때 삶이 선명해진다는 오래된 지혜(Memento Mori)를,
매일 쓸 수 있는 작은 도구로 옮긴 모바일 웹 앱입니다.

빌드 없이 `index.html` 단일 파일로 동작합니다. (모바일 우선, max-width 480px)

## 왜 만드는가 — 근거

| 원천 | 핵심 통찰 | 앱에서의 번역 |
|---|---|---|
| 메멘토 모리 (스토아 철학) | 죽음을 매일 상기하면 사소한 것에 덜 흔들린다 | **인생 달력** — 남은 시간을 주(週) 단위로 시각화 |
| 시한부 환자 연구 (죽음 현저성 → 긍정 정서·우선순위 재편) | 끝을 아는 사람은 관계와 의미에 시간을 쓴다 | **횟수로 보는 남은 시간** — "부모님을 1년에 2번 뵌다면, 앞으로 약 N번" |
| 브로니 웨어 『죽을 때 가장 후회하는 5가지』 | 죽어가는 이들의 후회는 놀랍도록 일정하다 | **후회 예방 체크인** — 5가지 후회를 오늘의 작은 실천으로 뒤집기 |
| 영화 〈코코〉 | 진짜 죽음은 아무도 기억하지 않을 때 온다 | **기억의 제단** — 떠난 이의 이름과 한 줄 기억을 남기는 공간 |
| 영화 〈죽은 시인의 사회〉 | Carpe Diem — 오늘을 붙잡아라 | **오늘의 물음** — 하루 하나, 삶을 정면으로 보게 하는 질문 |

## 페이지 구성

| 순서 | 역할 | 내용 |
|---|---|---|
| 히어로 | 후킹 | "당신에게 남은 계절을 세어본 적 있나요?" + 시작 CTA |
| 입력 | 개인화 | 생년월일 + 기대수명 슬라이더(기본 83세, 한국 기대수명) |
| 인생 달력 | 체감 | 1칸 = 1주. 살아온 주는 먹색, 남은 주는 여백 |
| 횟수의 언어 | 체감 | 남은 봄, 남은 보름달, 부모님을 뵐 횟수 등 "횟수"로 환산 |
| 오늘의 물음 | 습관 | 날짜 기반으로 매일 바뀌는 질문 1개 (카르페 디엠) |
| 후회 예방 | 실천 | 죽을 때 하는 후회 5가지 → 오늘 할 수 있는 작은 실천 체크 |
| 기억의 제단 | 연결 | 기억하고 싶은 사람의 이름 + 한 줄 기억 (localStorage) |
| 묘비명 | 마무리 | 내 묘비에 남길 한 줄 쓰기 → 저장 |

## 설계 원칙

- **겁주지 않는다.** 죽음은 공포 소재가 아니라 렌즈다. 카피는 담담하고 다정하게.
- **매일 1분.** 오늘의 물음 + 체크인 하나면 끝. 체류 시간을 늘리지 않는 것이 목표.
- **서버 없음.** 모든 데이터는 localStorage. 계정도, 수집도 없다.
- 디자인은 「여덟글자」의 손맛(한지·먹 감성)을 잇되, 밤하늘·촛불의 정적인 무드로 변주.

## 실행

```bash
npx http-server -p 8080
# http://localhost:8080/memento-mori/index.html 을 모바일 뷰포트로 열기
```

## 이후 로드맵 (프로토타입 이후)

1. **주간 리추얼** — 일요일 밤, 이번 주 인생 달력 1칸을 "채우며" 한 줄 회고 (알림은 PWA 전환 시)
2. **기억의 제단 공유** — 코코의 제단처럼 가족이 함께 채우는 공유 제단 (서버 필요, 첫 유료화 후보)
3. **유언장/버킷리스트 연동** — 묘비명 → 버킷리스트 → 실행 기록의 자연스러운 퍼널
