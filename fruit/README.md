# 🍑 과일사장님 — 과일 위탁판매 도우미

누구나 과일 위탁판매를 쉽고 편하게 할 수 있도록 돕는 서비스입니다.
토스·당근처럼 **한 화면에 한 가지 일**, 친근한 말투, 큰 타이포의 UI를 지향합니다.

- 별도 빌드 없이 정적 파일로 동작 (GitHub Pages 배포: `/fruit/`)
- 업로드한 엑셀은 **브라우저 안에서만** 처리 — 서버로 전송하지 않음 (SheetJS 로컬 번들)

## 기능 로드맵 (하나씩 완성)

| 단계 | 기능 | 상태 |
|---|---|---|
| 1 | **발주서 변환** — 쿠팡 발주서를 도매처별 발주서 엑셀로 분리 | ✅ 완료 |
| 2 | 송장 자동 입력 — 도매처 송장 파일을 읽어 쿠팡 송장 업로드 양식 생성 | 준비 중 |
| 3 | 썸네일·상세페이지 제작 | 준비 중 |
| 4 | CS 자동응답 | 준비 중 |
| 5 | 오늘의 판매 분석 — 옵션별 추이 | 준비 중 |

## 1단계: 발주서 변환 (`order.html`)

쿠팡윙 주문 목록 엑셀(.xlsx/.xls/.csv)을 올리면:

1. **자동 열 인식** — 앞쪽 15행에서 헤더 행을 찾고 수취인·전화·주소·옵션·수량 등을
   키워드 규칙으로 매핑. 잘못 읽으면 "열 매핑" 패널에서 직접 고칠 수 있음
2. **옵션 → 도매처 지정** — 옵션마다 도매처 이름을 적으면 `localStorage`에 기억되어
   다음 파일부터 자동으로 채워짐. 도매처가 하나뿐이면 건너뛰어도 됨
3. **다운로드** — 도매처별 개별 엑셀 또는 도매처별 시트로 묶은 통합 파일.
   출력 양식: 받는분 성명 · 전화번호 · 우편번호 · 주소 · 상품명(옵션) · 수량 · 배송메세지 · 쿠팡 주문번호

파일이 없어도 "샘플로 체험하기"로 전체 흐름을 확인할 수 있습니다.

## 구조

```
fruit/
  index.html          홈 (기능 허브)
  order.html          발주서 변환
  assets/
    style.css         디자인 시스템 (토스풍)
    order.js          발주서 변환 로직
    vendor/xlsx.full.min.js   SheetJS 0.18.5 (로컬 번들)
```

## 실행

```bash
npx http-server -p 8080

```
