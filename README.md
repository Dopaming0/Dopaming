# 여덟글자 四柱八字 — 사주명리 상담소 모바일 웹

사주전문가의 사주풀이 서비스를 위한 모바일 웹 인트로 페이지입니다.

## 콘셉트

- **겸재 정선의 진경산수화풍**: 한지 바탕, 먹으로 그린 산세(인왕제색도 무드), 산허리 안개, 인주 낙관
- **사람 냄새 나는 카피**: 전문가가 직접 쓴 편지, 후기, 삐뚜름한 도장과 종이 질감 등 손맛 위주의 디자인
- **타이포그래피**: 본문·제목은 Pretendard(자간 −2~−4%, 강조는 ExtraBold 800), 편지·후기·인용 등 감성 파트는 고운바탕(명조), 한자는 Song Myung
- 조선 관상감(觀象監)의 전문성·신뢰감을 이야기의 축으로 사용
- 모바일 우선(max-width 480px), 별도 빌드 없이 `index.html` 단일 파일로 동작

## 페이지 구성 (세일즈 퍼널)

카피는 메인 2줄, 서브 1~2줄 이내로 압축. 인트로가 전환의 80%라는 원칙에 따라 무료 맛보기를 최상단에 배치.

| 순서 | 역할 | 내용 |

| 맛보기 體感 | 리드 확보 | 생년월일·출생시각 → **명식표** 즉석 생성, 월주 봉인(封) 티저 |

## 명식 계산

- **일주**: 기준일 1949-10-01(갑자일) 앵커, 율리우스 적일(JDN) mod 60
- **년주**: 입춘(약 2월 4일) 이전 출생은 전년도로 보정
- **시주**: 오서둔시법 — 일간에 따라 자시의 천간을 정하고 지지 순서대로 진행. 자시 출생은 야자시 논쟁 안내
- **월주**: 절기력이 필요하므로 계산하지 않고 봉인 처리 → 전문가 상담 유도 장치

## 우리사이 — 커플 앱 (`couple/`)

연인이 매일 사용하며 관계가 깊어지도록 돕는 커플 앱입니다. 사주 페이지와 동일하게 빌드 없이 `couple/index.html` 단일 파일로 동작하며, 모바일 우선(max-width 480px)입니다.

- **오늘의 질문**: 106개 질문을 5단계 커리큘럼(가볍게 열기 → 서로 발견하기 → 마음 꺼내기 → 우리 그리기 → 깊이 만나기)으로 매일 하나씩. 아론의 '36가지 질문' 원리처럼 쓸수록 깊어지고, 서로 몰래 답한 뒤 **둘 다 답해야 공개** (핵심 루프)
- **반응 & 답글**: 공개된 상대의 답에 이모지 반응과 한 줄 답글을 남겨 대화로 잇기
- **기분 체크인**: 이모지로 오늘 기분 공유
- **고마워 한 줄**: 매일의 감사 표현 기록
- **추억 타임라인**: 데이트·일상 기록
- **기념일**: D+day, 100일 단위·주년 자동 계산 + 커스텀 기념일 D-day
- **주간 미션**: 매주 함께 완료하는 관계 미션 36종
- **애정 온도**: 최근 2주의 기록량으로 산출, 연속 문답 스트릭
- **마일스톤 선물** (목표: 더 오래 더 깊게 사랑하자): 함께 완성한 문답 일수 기준 3·7·14·28·50·100일 달성 시 선물 증정
  - 3일 보너스 질문 뽑기 → 7일 커플 테마 컬러 → 14일 미션 다시 뽑기 → 28일 우리의 첫 달 리포트(인쇄) → 50일 타임캡슐 편지 + 데이트 쿠폰북(인쇄) → 100일 3종 선물세트(연애문답지 인쇄 + 이별방지키트·굿즈 실물 신청 코드)
- 기본은 한 기기를 번갈아 쓰는 모델(사용자 전환 + 답변 잠금). 모든 기록은 localStorage, JSON 백업/복원 지원

### 두 기기 동기화 (커플 연결, 선택)

각자의 폰에서 실시간으로 함께 쓰려면 무료 Supabase 프로젝트 하나만 연결하면 됩니다.

1. [supabase.com](https://supabase.com)에서 무료 프로젝트 생성
2. SQL Editor에서 테이블 생성:
   ```sql
   create table couples (
     code text primary key,
     data jsonb not null,
     updated_at timestamptz not null default now()
   );
   alter table couples enable row level security;
   create policy "anon rw" on couples for all to anon using (true) with check (true);
   ```
3. `couple/index.html`의 `SYNC_CONFIG`에 프로젝트 URL과 anon key 입력
4. 앱 → 더보기 → **커플 연결**: 한 명이 코드를 만들고, 상대가 그 코드로 연결

동작 방식: 12초 폴링 + 저장 시 푸시, 사람별 선점·id 합집합 방식의 무손실 병합. 상대가 답을 남기면 토스트와 브라우저 알림(권한 허용 시)으로 알려줍니다. 연결 코드를 아는 사람은 데이터에 접근할 수 있는 MVP 구조이므로, 정식 서비스 전에는 Supabase Auth 기반 접근 제어로 교체가 필요합니다.
