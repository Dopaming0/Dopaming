# 우리의 계절 — 함께라서 소중한 시간

커플 앱 「우리사이」(`couple/`)와 메멘토 모리 앱 「남은 계절」(`memento-mori/`)을
하나로 합친 앱입니다. 빌드 없이 `index.html` 단일 파일로 동작합니다. (모바일 우선, max-width 480px)

## 합치기 콘셉트

두 앱의 교집합은 **"시간이 유한해서, 함께하는 오늘이 소중하다"** 입니다.

- **뼈대는 우리사이**: 파스텔 코랄의 귀여운 디자인, 문답·기분·추억·기념일·마일스톤 보상 등 매일 쓰는 커플 루프 전부 유지
- **남은 계절은 렌즈로**: 죽음·묘비 같은 어두운 소재는 걷어내고, "함께할 봄이 아직 49번 남았어요" 같은 다정한 언어로 번역해 **계절 탭** 하나에 담음
- 이름 「우리의 계절」 = 우리사이 + 남은 계절

## 새로 생긴 것 — 계절 탭 🌸

| 원본 (남은 계절) | 커플 버전 (계절 탭) |
|---|---|
| 인생 달력 (1칸=1주, 먹색) | **우리의 시간 달력** — 함께한 주는 코랄, 이번 주는 골드, 남은 주는 여백 |
| 횟수의 언어 (남은 봄, 보름달…) | **함께 맞이할 봄 · 나란히 볼 보름달 · 같이 뒹굴 주말 · 여행 · 생일 · 첫눈** |
| 오늘의 물음 (카르페 디엠 30문) | **커플 카르페 디엠 30문** — "오늘이 함께하는 마지막 하루라면?" |
| 후회 예방 체크인 (죽기 전 후회 5가지) | **후회 제로 연습** — 오래된 연인들의 후회 5가지를 오늘의 실천으로 |
| 묘비명 쓰기 | **OUR LAST LINE** — "우리 이야기가 책이 된다면 마지막 문장은?" |
| (로드맵의 버킷리스트) | **우리 버킷리스트** — '언젠가'를 '조만간'으로 바꾸는 목록 |
| 기억의 제단 | 커플 앱 정서와 맞지 않아 제외 (원하면 추가 가능) |

홈 화면에는 매일 바뀌는 티저 카드가 하나 추가됩니다:
"🌸 함께 맞이할 봄 — 앞으로 약 49번 · 그러니까 오늘, 조금 더 다정하게"

## 데이터

- 두 사람의 생일 + 기대수명(기본 83세)은 계절 탭에서 입력, `localStorage`에만 저장
- 저장 키는 우리사이와 같은 `woorisai.v1` — 기존 우리사이 사용자의 기록(문답·추억 등)이 그대로 이어집니다
- 두 기기 동기화(Supabase)·백업/복원도 계절 탭 데이터(생일·후회체크·버킷·마지막 문장)까지 무손실 병합

## v2.0 — 음성 우선 (텍스트 → 목소리)

매일 주고받는 것(**오늘의 질문 답 · 고마워 한 줄 · 기분**)을 30초 목소리로 바꾸고,
말한 내용을 **자동으로 받아 적어** 글로도 남긴다. 추억·버킷리스트·기념일은 글 그대로 — 책과 검색을 지키기 위해.

### 설계의 핵심 결정 5가지

| # | 결정 | 이유 |
|---|---|---|
| 1 | `answers[k].p1/p2`는 **영원히 문자열** | 객체로 바꾸면 `esc()`가 `[object Object]`를 찍는 곳이 인쇄물 포함 수십 곳. 목소리는 형제 필드 `vo`에 **조각 id로만** 붙인다 |
| 2 | 오디오 실체는 **`S.voices`에만** | 저장공간 방어·자동백업 제외·전송량 축소가 전부 이 배열 하나만 보고 동작한다. answers 안에 넣는 순간 네 방어장치가 한꺼번에 무너진다 |
| 3 | 받아쓰기 실패해도 **가짜 글을 넣지 않는다** | 게이트는 `hasAns()`(글 **또는** 목소리)로 판정하고, 책·아카이브는 `bookText()`가 「그날은 목소리로만 남겼어요」로 렌더한다. 빈 줄 인쇄도, 플레이스홀더 오염도 피한다 |
| 4 | 만료는 **덜 아까운 것부터** | ① 글 있고 상대도 들음 → ② 글은 있음 → ③ **글이 없는 것은 마지막까지** + 방금 녹음한 것은 무조건 보호 |
| 5 | 30초는 **상한이지 목표가 아니다** | 8초에 「이 정도면 충분해요 🫶」가 뜨고 정지 버튼이 「그만두기」→「다 말했어요」로 바뀐다. 짧게 말할수록 오래 보관된다 |

### 화면

- **📻 오늘의 라디오** (홈 최상단) — 상대가 남긴 오늘의 소리를 편성표로 보여주고 ▶ 하나로 이어 듣는다.
  - **잠긴 트랙**: 내가 아직 답하지 않은 질문은 🔒로 뜨고 **오디오 주소를 DOM에 아예 싣지 않는다.**
    정확한 길이도 숨기고(「길게 남겨뒀어요」) 답한 뒤에 공개 — 2인 앱에서 「0:24」는 곧 「저만큼 말해야 하나」가 된다
  - **🔇 소리 없이 보기** — 모든 트랙이 받아쓴 글로 바뀐다. 회의·지하철·아이 재우는 밤에도 마음은 도착한다 (잠긴 트랙의 글은 새지 않는다)
- **🎙 녹음 시트** (질문·고마워·기분 공용) — 말문 트기 첫마디 칩 → 게이지(카운트다운 아님) → **미리듣기 + 받아쓴 글 수정** →
  [보내기] / [다시 녹음] / **[✏️ 글만 보내기]** / [지우기]. 오타가 실제로 고쳐지는 유일한 창이자, 부끄러움의 해독제
- **✍️ 아직 글로 못 옮긴 목소리 N개** — Web Speech API는 **저장된 파일을 다시 받아쓰지 못한다.**
  녹음 순간 인식이 실패하면 사람이 옮겨 적는 게 유일한 길이라, 조용히 두지 않고 홈에서 한 줄 적을 자리를 연다
- **기분**은 이모지 1탭 그대로. 고른 뒤에만 「🎙 왜 그런지 한마디 (선택)」이 조용히 뜬다 (동작 수는 늘지 않는다)

### 용량

μ-law(G.711) 전환으로 **초당 20.8KB → 10.4KB (정확히 절반)**. 30초 = 314KB.
fmt 태그 7 · 8bit · fact 청크 포함한 규격대로 만들어 `<audio>` 재생·`decodeAudioData` 모두 확인, 실측 SNR 37dB(전화 음질).
기존 16bit 조각은 WAV 헤더가 자기설명적이라 그대로 재생된다.
녹음은 **들어오는 즉시 8kHz로 줄여 담는다** — 30초 원음을 통째로 쌓으면 5.7MB + 사본이 잠깐 떠서 구형 아이폰 탭이 죽을 수 있다.

### ⚠️ 같이 고친 것 — 동기화 무한 핑퐁 (v1.6부터 있던 실제 버그)

고마워는 `unshift`, 추억은 `push`로 쌓여 **두 기기의 배열 순서가 영영 달라지는데**,
`syncPull` 끝에서 JSON을 통째로 비교해 다르면 push하므로 — 내용이 같아도 순서만 달라
**12초마다 서로를 밀어내는 무한 동기화**가 성립한다(브라우저에서 재현 확인).
`exportable()`에서 목록을 id로 정렬해 두 기기의 JSON을 글자까지 같게 만들어 해소.
함께: PATCH에 `select=updated_at`을 붙여 응답 왕복량 절반, 스키마 버전(`ver`) 게이트로 구버전 폰이 새 기록을 삼켜 지우는 사고 차단.

검증: 구버전 데이터 호환 · 잠긴 트랙 주소 미노출/길이 은닉 · 목소리로만 답해도 게이트/스트릭/통계 정상 ·
목소리로 답한 상대를 두고 질문 넘기기 잠김 · 책과 아카이브의 「목소리로만 남겼어요」 · 「가장 길게 나눈 날」 오염 없음 ·
보관 우선순위(글 없는 것·미청취·방금 녹음한 것 보호) · 저장소 100% 상태에서 글·사진·문답 생존 · 전 탭 회귀 · 콘솔 에러 0.
**받아쓰기 자체는 이 PC에 마이크가 없어 검증하지 못했다 — 실기기 확인 필요.**

## v1.8 — 🎙 목소리 한 조각 + 콩닥이 솔로 크레딧

**🎙 목소리 한 조각** (추억 탭 → 목소리)
- 15초짜리 음성. 떨어져 지내는 밤에 글로는 안 담기는 것을 담는다
- **기기 호환 문제를 원천 차단**: 사파리는 mp4, 크롬은 webm으로 녹음돼 그대로 저장하면 상대 기기에서 안 열린다.
  그래서 MediaRecorder를 쓰지 않고 마이크 소리(Float32)를 직접 받아 **8kHz 모노 WAV**로 담는다 —
  어느 기기에서나 열리고, 15초에 313KB로 작다. 녹음이 조용하면 자동으로 볼륨을 맞춰준다(최대 6배)
- 상대의 새 목소리는 **홈 카드 + 알림**으로 도착하고, 들으면 카드가 조용히 사라진다(들었는지는 기기별로만 기록)
- **보관 규칙**: 용량 한도(1.2MB) 안에서 최근 것부터 보관 — 짧은 조각은 20개 가까이, 15초 꽉 채운 조각은 3~4개.
  최근 2개는 한도를 넘겨도 지킨다. 오래 간직할 조각은 **저장** 버튼으로 .wav 파일 내려받기
- **동기화는 최근 7일 · 3개까지만** 전송 (전부 실어 보내면 저장할 때마다 몇 MB를 올려야 한다).
  양쪽이 같은 규칙으로 고르므로 합쳐진 결과는 서로 같아진다
- **저장 공간이 가득 차면** 자동백업 → 오래된 목소리 순으로 자리를 비우고, **글과 사진은 끝까지 지킨다**.
  자동백업 스냅샷에도 목소리를 복사하지 않는다(복원해도 지금 목소리는 그대로)

**🐣 콩닥이 솔로 크레딧**
- 지금까지 콩닥이는 **둘 다** 채운 것만 먹었다 → 상대가 바쁜 계절엔 굶었다
- 이제 **한쪽만 답한 날과 목소리 한 조각도 반 그릇**. *"혼자 남긴 마음 N번도 반 그릇씩 챙겨뒀어요"*
- 리서치 결론에 따라 **"누가 안 했는지"는 절대 세지 않는다** — 격려만 남기고 비난은 만들지 않는다
- 목소리는 보관에서 밀려나도 밥은 남는다(사람별 누적 카운터 `voiceFed`)

**📕 실물 책으로 만들기** (더보기 → 우리의 이야기 책)
- 만쥬님 아이디어였던 실물 제본. PDF 저장 → A5 용지 → 제본 업체 → 사진 해상도 주의까지 4단계 안내

검증: 녹음 전 과정(가짜 마이크 스트림)·15초 자동종료·1초 미만 거절·그만두기·마이크 거부·재생/일시정지/전환·
재생 중 삭제·용량한도 정리(25→18개)·최소 2개 보장·전송범위(5개 중 3개, 188KB)·병합 합집합/멱등·
저장소 100% 채운 상태에서 목소리만 비우고 글·사진·문답 생존·자동백업 1KB 유지·구버전 데이터 호환. 콘솔 에러 0.

## v1.7 — 🐣 다른 질문 주세요 + 기록 지키기

**🐣 다른 질문 주세요** (리서치 가성비 1위 — 15개 앱에 33건 요구)
- 오늘의 질문이 안 맞을 때 넘길 수 있는 유일한 출구. 하루 2회, 상대가 이미 답한 뒤에는 잠김
- '스킵'이 아니라 **콩닥이가 대신 물어다 주는** 프레이밍
- **★ 우리만 가능한 반전 — 「미뤄둔 질문」**: 다른 앱은 넘긴 질문을 영영 버리지만(Agapé는 서버 프로필에 안 남김),
  우리는 기기에 다 남으므로 **90일 뒤 조용히 되돌려준다** — *"그때 미뤄뒀던 질문이에요. 지금은 어때요?"*
  되돌아온 질문을 다시 미루면 커리큘럼은 건드리지 않고 90일만 더 밀린다
- 넘긴 만큼 커리큘럼이 앞으로 나아가(`skipCountUpTo`) 다음날 같은 질문이 다시 나오지 않는다. 동기화는 합집합 병합

**🛡 기록 지키기** (1차 리서치 1순위 니즈 = 데이터 영속성에 대한 대응)
- `navigator.storage.persist()` 요청 — 브라우저가 저장을 임의로 비우지 못하게
- 미설치 상태면 홈 카드에서 **데이터 소실 위험을 명시**하고 설치 안내 (숨김은 영구가 아니라 14일 스누즈)
- 더보기 → 🛟 기록 지키기: 보호 상태(설치됨/부분/위험), 사용량, **마지막 내보내기 백업 경과일**
- ※ 사파리는 홈 화면 미설치 시 일정 기간 방문이 없으면 저장을 비우므로, 설치가 사실상 유일한 보호책
